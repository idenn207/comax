# Product

## Register

product

## Users

Comax Secrets 서버를 직접 운영하는 소수의 엔지니어. 두 갈래가 같은 화면을 공유한다.

- **CLI 능숙형**: 평소엔 `secret push / pull / diff` 로 일한다. GUI는 audit 추적·환경 비교·롤백처럼 흐름을 한눈에 보고 싶을 때 연다. 사용 빈도는 낮지만 한 번의 실수가 사고가 되므로 "행동을 확신할 수 있는 신호"가 결정적이다.
- **GUI 우선형**: CLI 명령어를 외우지 않고도 같은 일을 한다. 신규 envvar 추가·환경 비교·롤백·audit 추적을 모두 마우스 + 키보드 단축키로 완주한다. 이 도구의 가치 절반은 "CLI 러닝커브 없이도 같은 안전성"에 있다. 명령어 이름을 모르는 운영자가 화면만 보고 다음 액션을 짐작할 수 있어야 한다.

두 갈래 모두 운영자 위치다. 데스크탑 앱 포팅이 예정되어 있어 OS 셸 안에서 자연스럽게 어울리는 정체성이 요구된다.

## Product Purpose

운영자가 자신의 인프라 안에서 시크릿을 안전하게 보관하고, 환경 간 일관성을 유지하고, 변경을 추적·롤백할 수 있게 한다. 클라우드 SaaS 시크릿 매니저의 편의성을 self-hosted 환경에서 재현하되, "내가 직접 돌리고 있다"는 통제감을 화면에서도 그대로 전달한다.

성공 신호:

- 운영자가 GUI를 열어 90초 안에 "지금 상태가 안전한가"를 판단할 수 있다.
- 환경 비교 화면에서 누락·드리프트가 화면을 스크롤하지 않고 보인다.
- 시크릿 값이 실수로 노출·로깅되지 않는다 (UI에서도 마스킹 강제).

## Brand Personality

세 단어: **조용함, 정직함, 직관적**.

- **조용함**: 색·모션·문장으로 운영자를 놀라게 하지 않는다. 시그널은 시그널일 때만 켜진다. 발화는 칭찬·과장 없이 사실. "12개 키가 변경되었습니다" 같은 평서문.
- **정직함**: 시스템 상태(시크릿 가림 / 환경 누락 / 롤백 직후)는 가리지 않고 그대로 보여준다. "잘 된 것 같아요" 같은 모호한 토스트 금지. 위험과 비가역성은 라벨에서 미리 보인다.
- **직관적**: CLI 명령어를 모르는 운영자도 화면만 보고 다음 액션을 짐작할 수 있다. 발명한 인터랙션 없이, 운영 도구의 표준 어휘(list · table · timeline · breadcrumb · right-side action)를 정확히 빌려온다.

**핵심 레퍼런스: GitHub.** Issues / PR / Actions / Settings 화면이 가장 가까운 어휘 — 일러스트 없는 데이터 중심, 헤더 우측 액션, breadcrumb + tab, diff·activity feed 1급, monochrome 표면 위 작은 색 액센트는 의미에만. PR diff / commit timeline / audit log 의 GitHub 식 정직함이 이 대시보드의 시각적 정체성이다. Linear 의 단축키·command palette·drawer 패턴을 보조 어휘로 빌린다.

**톤**: UI 라벨·설명은 한국어 평어, 시스템 / 에러 메시지는 영어(운영자가 grep 가능해야 함). 군더더기 없는 명사 + 동사. 시각적 감정은 22시의 정돈된 작업 보드, 마케팅 deck 이 아니다.

## Anti-references

- **AWS / GCP 콘솔류**: 회색 위 회색, 끝없는 nav, 의미 없는 아이콘 그리드. "엔터프라이즈라서 못생겨도 된다"는 면죄부.
- **Doppler / Linear-copy SaaS 다크**: 인디고·바이올렛 액센트 + 다크 표면 + 둥근 카드 그리드. 같은 카테고리의 모든 도구가 머무는 첫 번째 reflex 레인.
- **Notion / Linear 그대로 복사**: 에디터형 도구의 미학을 운영 도구에 그대로 이식하는 것. 잘 따라하면 좋지만, 흉내가 들키면 정체성이 더 빈약해진다.
- **Vercel / shadcn 템플릿 룩**: 흰 배경, 미묘한 그림자, 라운드 카드 반복. "본 적 있다" 틱.
- **터미널 미학을 GUI로 그대로 이식**: monospace 도배, 검은 배경, `$` 프롬프트 흉내, CRT 효과. CLI 학습 없이 GUI로 같은 일을 한다는 가치가 무너진다. 진짜 터미널이 필요한 운영자는 진짜 터미널을 켠다. 이 대시보드는 그 반대 정체성이다.

데스크탑 앱 포팅 예정이므로 viewport 100vw에 기대는 fluid 디자인, browser-only 패턴, web font 종속도 함께 피한다.

## Design Principles

운영 결정의 5가지 원칙. 시각 규칙(픽셀·OKLCH)이 아니라 의사결정 가드레일. 신규 화면·컴포넌트·copy 결정 시 이 5개에 비추어 본다.

1. **색은 의미에만, 신호는 굵기·표면·간격으로** — chromatic accent 로 primary / current / selected 를 표현하지 않는다. 표면 elevation + border-strong + weight 600 이 그 자리를 대신한다. 색은 상태(success / danger / warning / info) 와 키보드 포커스(blue 250) 에만 산다.
2. **시크릿은 기본 가림, 노출은 의식적으로** — 시크릿 값은 항상 마스킹 상태로 시작한다. reveal 은 짧고 그 자체로 audit 흔적이 남는다. 클립보드 복사는 화면 노출 없이 가능해야 한다. 시크릿 값은 ≥5.5:1 대비를 유지하고, 로그 / 스크린샷 친화 텍스트로 새지 않는다.
3. **"누락"은 1급 시각 신호** — 환경 diff 에서 빠진 키는 일반 warning 이 아니라 "누락 전용" 시각을 가진다(행 배경 · 전용 chip · 정렬 우선). "없음" 이 아니라 "있어야 하는데 없음" 임이 한눈에 읽혀야 한다. 이 도구가 존재하는 이유다.
4. **운영자는 능숙하다고 가정하지만, CLI 사전 지식은 가정하지 않는다** — 강제 onboarding / tour / wizard 없음. 빈 상태는 다음 액션 1개만 제시한다. 액션은 그 액션이 일어나는 섹션 헤더에 둔다(top bar 는 글로벌 search · theme 만). 비가역 액션(rollback · delete) 은 명시 confirm + 결과 토스트, 자동 실행 금지. GUI 우선형 운영자가 화면만 보고 다음 액션을 짐작할 수 있어야 한다.
5. **외부 라이브러리 기본값은 신뢰하지 않는다** — Radix / Tailwind 토큰은 우리 monochrome 시스템으로 강제 매핑하고, 충돌 시 직접 셀렉터로 덮어쓴다. 시각 결정은 1920×1080 Playwright 회귀로 봉인되며, 변경은 검증 + 사용자 승인 후 풀린다. 동결 정책 자체가 디자인 결정이다.

## Accessibility & Inclusion

- WCAG 2.2 AA를 최저선으로 하되, 시크릿 값처럼 위험한 정보는 contrast를 한 단계 더 올린다 (≥ 5.5:1).
- 모든 인터랙티브 요소는 default / hover / focus-visible / active / disabled / loading / error 상태를 모두 갖는다.
- 키보드만으로 모든 흐름(검색 → 환경 진입 → 시크릿 회전 → 비교 → 롤백)을 완주할 수 있어야 한다.
- `prefers-reduced-motion`을 전역 토큰으로 존중하고, 시크릿 마스킹·복사·회전 같은 상태 전환은 모션 없이도 시각적으로 명확하게 보인다.
- 색상만으로 상태를 전달하지 않는다 (변경/누락/실패는 항상 아이콘·라벨·텍스트로 함께 표현).
